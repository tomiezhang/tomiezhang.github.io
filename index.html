service worker探究系列(一)：小实验，让hexo博客程序本机支持https

Veröffentlicht am 2016-12-12

最近看到谷歌开放了中国区域的开发者网站developers.google.cn，无需翻墙就能看到很多的谷歌技术，于是看了一下自己关心的技术领域，发现在web基础这块，谷歌在搞推送通知(web Push Notifications)，这块一直是我比较关心的领域，仔细阅读了下，发现还是基于service workers来的，service workers其实2014年就推出了草案，之前推的是一个叫App cache的API,基本上你可以理解service workers是一个浏览器代理服务，我们可以用它来做很多事情，比如后台消息传递、网络代理、转发请求、伪造响应（在线版的fidder[微笑脸]）、消息推送[这个后面研究]……等等。之前有人翻译了一篇介绍它原理的趣文，有兴趣可以看下service workers，你到底是个什么东西？

基本上，可以用一张图来了解sw:

##service workers的生命周期

  2. 激活，激活成功后，打开chrome://inspect/#service-workers 可以查看当前运行的SW任务

详细的SW研究后面再写，有点跑题跑远了，service workers有一个要求，网站必须是支持https的，http下是无法使用sw服务的，加上IOS即将对所有APP中内置的页面强制使用https，这就导致我们必须在本地开发环境中，先把https服务架起来，那该怎么搞？

如何在本机架设https?

由于前端开发，所以本地全部都是Node环境，自己博客也是基于Node的hexo程序，目标是让博客支持SW服务，那具体来说就是如何在本地NODE服务器上添加https支持，本着深挖学习的精神，我们得先了解下https都是个啥？

Https是个啥？

HTTPS (基于安全套接字层的超文本传输协议 或者是 HTTP over SSL) 是一个 Netscape 开发的 Web 协议。

你也可以说：HTTPS = HTTP + SSL

HTTPS 在 HTTP 应用层的基础上使用安全套接字层作为子层。

为什么需要Https?

我们知道，Http是使用请求/响应的过程，因此信息可在服务器间快速、轻松而且精确的进行传输。当你访问 Web 页面的时候你就是在使用 HTTP 协议，但 HTTP 是不安全的，可以轻松对窃听你跟 Web 服务器之间的数据传输。在很多情况下，客户和服务器之间传输的是敏感歇息，需要防止未经授权的访问。为了满足这个要求，HTTPS应运而生。

HTTPS与HTTPS有什么不同？

  1. URL开头是以Https开始
  2. 默认端口为443
  3. Https工作在传输层
  4. 需要证书

而了解HTTPS协议，必须要了解SSL传输协议，它是实现HTTPS的关键，概念就不用解释了，一张图就可以说明：

SSL位于HTTP和TCP之间，它肩负着加密的使命

而整个SSL的通信过程，我觉得可以看下阮一峰老师的一篇博客，非常形象,这里摘录部分：

假定客户端叫做爱丽丝，服务器叫做鲍勃，整个握手过程可以用下图说明

第一步，爱丽丝给出协议版本号、一个客户端生成的随机数（Client random），以及客户端支持的加密方法。

第二步，鲍勃确认双方使用的加密方法，并给出数字证书、以及一个服务器生成的随机数（Server random）。

第三步，爱丽丝确认数字证书有效，然后生成一个新的随机数（Premaster secret），并使用数字证书中的公钥，加密这个随机数，发给鲍勃。

第四步，鲍勃使用自己的私钥，获取爱丽丝发来的随机数（即Premaster secret）。

第五步，爱丽丝和鲍勃根据约定的加密方法，使用前面的三个随机数，生成”对话密钥”（session key），用来加密接下来的整个对话过程。

以上，通过这个原理学习，我们发现，想在本地搭建HTTPS，必须得先有个证书，那么该怎么操作呢？

开始搭建基于NODE SERVER的HTTPS服务

简而言之，我们需要SSL证书，事实上有两种证书：一种是由‘Certificate Authority’【权威认证】签发的，或者也叫CA；另外一种是由‘self-signed certificates’【私有证书】来生成的。使用权威认证签发的证书，会告诉你用户的浏览器，这个网站是可信的安全的，对于正式产品线上环境来说，这个肯定是首选；而私有证书则是部署测试环境的最佳选择，因为在localhost嘛，所以就选私有就好了。

想要创建私有证书，在你的shell里运行以下命令：

openssl genrsa -out key.pem
openssl req -new -key key.pem -out csr.pem
openssl x509 -req -days 9999 -in csr.pem -signkey key.pem -out cert.pem
rm csr.pem

你会得到两个文件，一个是cert.pem(证书)，一个是key.pem（私钥）。这就是你所需要的所有SSL证书了。使用NODE的的话，可以快速搞个hello world了：

var https = require('https');
var fs = require('fs');

var options = {
  key: fs.readFileSync('key.pem'),
  cert: fs.readFileSync('cert.pem')
};

var a = https.createServer(options, function (req, res) {
  res.writeHead(200);
  res.end("hello world\n");
}).listen(8000);

主要区别是HTTP和HHTPS模块的传参不同。

注意：这里使用的fs.readFileSync来加载证书的，而没有使用fs.readFile，这是有原因的，前者会异步的加载文件，而不是同步，如果你使用后者，而且你的网站流量还很大，那估计就会悲剧了（阻塞问题会要命）

OK，现在服务就起来了，跑下看看呗：

https://localhost:8000
